The protocol

Three phases. One human in the loop.

Anthropic built MCP to connect models to the world. Rosewood proposes HAP to connect guests to hospitality. It rides on MCP, adds the semantics of a stay, and treats the guest's authorization with the rigor of OAuth — not the looseness of a CRM checkbox.

01
The human approves scope

Handshake

A scope-bounded session opens. The guest authorizes specific categories — lodging, dietary, visit purpose — for a bounded TTL. A signed consent_token is minted. No data has moved yet.

02
Agents negotiate · no input from the human

Agent-to-Agent

Within the authorized scope, the guest's agent and the property's concierge agent exchange the actual preferences and produce an arrival orchestration. The human watches. They don't type.

03
The human confirms outcome

Confirmation

The human reviews what the agents agreed on — the staff brief, the dietary flags, the trip mode. One tap commits the stay. At checkout the plugin disconnects, the memory returns to the guest's agent, the property keeps nothing.

The five tools

HAP is an MCP plugin.
Five tools, one open protocol.

Install once — like Google Drive, GitHub, Calendar. A session opens per stay and auto-disconnects at checkout. The same mental model your Claude already knows.

hap_handshake
Mints a signed consent token for a scope and a TTL. Opens the session. Writes to the hash-chained audit log.
returns · session_id · consent_token · audit_url
hap_propose_arrival
Negotiates an arrival orchestration with the concierge agent within authorized scope. Produces a staff brief and a voice line.
returns · staff_brief · flow_profile · voice_line
hap_in_stay_action
Routes in-stay intents. Complaints, maintenance and minors check-in always escalate to a human per the inviolable rules of HAP §12.
returns · escalation · staff_brief · guest_response
hap_post_stay_memory
Returns the refined memory snapshot to the guest's agent. The property keeps nothing — only the operational signal, pseudonymized.
returns · memory_snapshot · retention_confirmation
hap_generate_dispute_brief
Generates a WARDEN-signed timeline of a stay from the immutable audit. Reputation defense with cryptographic evidence.
returns · brief_markdown · signature · timeline
hap_checkout
Two-step: invoice preview with billing split by trip flow (Bleisure splits corp ↔ personal) then revokes the session, returns memory, emits the Departing Threads chain.
returns · revoked_session_ids · invoice_summary

Trust by construction

Five guarantees,
not five marketing promises.

HAP's privacy posture is not a privacy policy — it's an architecture. Every guarantee below is enforced at the protocol layer or invalidates compliance.

  • ScopeThe guest authorizes categories one by one. Anything outside scope is blocked at the tool boundary.
  • TTLEvery session expires. Default 72 h. The plugin disconnects on its own.
  • Zero retentionThe property queries on demand. It never stores a copy of guest data.
  • AuditEvery event is hash-chained with SHA-256. Tampering is detectable and immediate.
  • RevocationThe guest revokes scope from their agent at any time. The protocol propagates within seconds.
  • Human in the loopComplaints, maintenance, minors, bereavement — every inviolable rule of HAP §12 routes to a human, not an agent.
